# 水文模型率定AI智能体 - 任务指导书

## 一、项目目标

设计并实现一个**AI驱动的水文模型率定系统**，让AI智能体（LLM）在水文模型率定流程中发挥核心作用。

**核心问题**：如何让AI智能体在水文率定中真正发挥作用，而不是沦为"语音助手"？

## 四、最小功能范围（三场洪水版本）

实现以下完整流程：

```
1. 上传3场洪水数据文件（CSV） → 2. 配置列名映射 → 
3. 设置流域面积 → 4. 设置模型参数范围 → 
5. 运行率定 → 6. 查看结果报告 → 7. 查看过程线图
```

**简化点**：
- 只实现一个新安江模型（其他模型暂不做）
- 只实现一种优化算法（DE即可）
- 不需要多文件批量处理，一次只处理一个文件（包含3场洪水）
- 不需要上游汇流演算
- 不需要BMA集合预报

## 七、关键引导问题

在设计系统时，始终思考：

> **问题1**：这个环节如果没有AI，人类专家会怎么做？
> - 如果人类也做不了，那AI也做不了
> - 如果人类能做但做得慢，AI可以加速
> - 如果人类能做但需要查很多资料，AI有优势

> **问题2**：AI的输出如何验证？
> - AI可能一本正经地胡说八道
> - 必须设计验证环节，而非完全信任AI

> **问题3**：系统的"护城河"是什么？
> - 不是AI调用（大家都差不多）
> - 而是领域知识嵌入、业务流程设计、用户交互体验

## 八、注意事项

1. **不要照抄源码**：参考案例只作为功能理解，不要复制代码

3. **重视验证**：AI输出必须可验证，不能盲目信任
