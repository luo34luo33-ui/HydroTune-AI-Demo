# XGBoost误差校正功能说明

## 一、概述

XGBoost误差校正是对水文模型模拟结果进行后处理校正的功能，通过机器学习方法预测模型误差并修正模拟结果，提升模拟精度。

## 二、使用方法

### 1. 启动应用

```bash
streamlit run app.py
```

### 2. 选择上传模式

在侧边栏选择 **"多文件（每文件一场洪水）"** 上传模式。

### 3. 上传数据文件

上传多个洪水数据文件（支持CSV/Excel格式），每行包含：
- 时间
- 降水 (precip)
- 蒸发 (evap)
- 流量 (flow)
- 可选：上游流量 (upstream)

### 4. 配置率定参数

- 选择时间步长（daily/hourly）
- 设置预热期
- 选择要率定的水文模型（HBV、新安江、水箱）
- 设置优化算法和迭代次数

### 5. 开始分析

点击 **"🚀 开始分析"** 按钮，系统将：
1. 自动划分训练/测试场次（75%:25%）
2. 率定各水文模型参数
3. **自动进行XGB误差校正**

### 6. 查看结果

率定完成后，在 **"🔧 XGBoost 误差校正"** 区域查看：
- 校正前后NSE对比
- 流量过程线对比图
- 可下载校正结果CSV

## 三、技术细节

### 3.1 特征工程

输入特征（共8个）：

| 特征 | 描述 |
|------|------|
| e(t-1) ~ e(t-5) | 前5个时间步的真实误差 |
| p(t-1) ~ p(t-3) | 前3个时间步的降水量 |

**注意**：校正时使用**真实误差**（flow - simulated）构建特征，而非XGB预测的误差递归。

### 3.2 训练集划分

- **固定随机种子**：seed=42，保证可复现
- **按场次划分**：80%场次训练，20%场次测试
- **统一模型**：所有测试场次共用同一个XGB模型

### 3.3 XGB模型参数

```python
XGB_PARAMS = {
    'objective': 'reg:squarederror',
    'max_depth': 10,
    'learning_rate': 0.05,
    'n_estimators': 300,
}
```

### 3.4 校正公式

```
error(t) = flow(t) - simulated(t)
corrected(t) = simulated(t) - predicted_error(t)
```

其中 predicted_error(t) 由XGB模型预测，特征为 [e(t-1), ..., e(t-5), p(t-1), ..., p(t-3)]

## 五、注意事项

1. **数据量要求**：至少需要3个洪水场次才能进行XGB误差校正
2. **预热期处理**：预热期数据会被剔除，不参与训练和评估
3. **上游流量**：如果启用上游汇流，会在模型运行后叠加
4. **NSE计算**：预热期数据不参与NSE计算

## 六、常见问题

### Q1: 为什么校正效果不好？

可能原因：
1. 水文模型误差模式复杂，8个特征不足以捕捉
2. 训练场次太少（<10场）
3. 特征设计不适合该流域特性

### Q2: 如何改进校正效果？

1. 增加更多特征（模拟流量滞后、累积降水等）
2. 调整XGB参数（降低max_depth、增加正则化）
3. 尝试线性回归模型

### Q3: 校正后NSE反而下降？

说明XGB模型预测的误差反而引入了额外误差，可能原因：
- 训练集和测试集分布差异大
- XGB过拟合
- 特征不足
