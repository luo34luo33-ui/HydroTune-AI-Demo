# 洪水语义编码工作流程

## 概述

洪水语义编码是HydroTune-AI系统的核心模块之一，旨在将原始水文数据转换为结构化的"事件+阶段+特征+语义描述"的多模态表示，供LLM进行智能分析。

## 工作流程

### 1. 数据质量评估
- **数据完整率**：检测缺失值比例
- **时间连续性**：检查时间间隔是否一致
- **降水-径流相关**：计算降水与径流的相关系数
- **质量等级**：综合评估数据质量（A/B/C/D）

### 2. 洪水事件检测
- 自动识别流量序列中的洪水起涨点
- 确定洪水结束点（退水至基流）
- 定位洪峰位置
- 支持单文件多场洪水和多文件模式

### 3. 语义编码生成
对每场洪水提取以下特征：

| 特征 | 描述 |
|------|------|
| event_id | 洪水编号（基于日期） |
| magnitude | 量级（small/medium/large/extreme） |
| peak_flow | 峰值流量 (m³/s) |
| flood_volume | 洪水总量 (10⁴m³) |
| duration | 洪水历时 |
| rise_type | 涨洪类型（slow/moderate/rapid） |
| recession_type | 退水类型（slow/moderate/rapid） |
| peak_shape | 峰型（rounded/sharp/multi-peak） |
| rainfall_driven | 降雨驱动强度（low/medium/high） |
| representative_score | 代表性评分（0-100） |
| selection_reason | 选取理由说明 |
| baseflow | 基流 |
| peak_ratio | 峰基比 |
| total_precip | 总降水量 |

### 4. 可视化分析图
- 绘制流量过程线与降水过程
- 标注各场洪水边界
- 标记洪峰位置
- 基流分离可视化

### 5. LLM智能分析报告
- 调用大语言模型生成专业分析报告
- 解读洪水特征规律
- 评估数据质量
- 提供场次选取建议

## 核心模块

- `FloodSemanticEncoder`: 洪水语义编码器
- `SemanticPreAnalyzer`: LLM智能预分析器
- `FloodSemanticCode`: 语义编码数据类

## 输出格式

语义编码以JSON格式供LLM使用，包含：
- 场次标识与分类
- 定量特征（峰值、洪量、历时等）
- 定性描述（涨落段类型、峰型等）
- 代表性评分与选取理由
